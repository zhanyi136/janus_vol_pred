## 3. 特征工程

### 3.1 设计原则

- **只用 book_ticker 数据**：计算波动率相关因子
- **Numba加速**：所有计算函数用Numba实现
- **防未来函数**：采样时严格 `timestamp < current_time`

### 3.2 20ms采样规则（防止未来函数）

```
当前时间 = T

选择条件：
1. timestamp < T  (严格小于，不能等于)
2. local_timestamp < T  (严格小于)
3. 如果有多个相同timestamp → 选最大的local_timestamp

这样保证：任何时刻用的数据都是"截至当前T之前最新可获得的数据"
```

```python
def sample_20ms(df: pd.DataFrame, current_time_ns: int) -> Optional[pd.Series]:
    """
    对于当前时间T，选取满足条件的最优数据点
    """
    valid = df[
        (df['timestamp'] < current_time_ns) &
        (df['local_timestamp'] < current_time_ns)
    ]
    if valid.empty:
        return None
    # 如果有多个相同timestamp，取最大的local_timestamp
    valid = valid.loc[valid.groupby('timestamp')['local_timestamp'].idxmax()]
    # 取timestamp最大（即最新）的记录
    return valid.loc[valid['timestamp'].idxmax()]
```

### 3.4 波动率计算流程

```

    │
    ▼

    │
    ▼

    │
    ▼
滚动窗口计算已实现波动率
vol_{N}m = std(mid_price_chg[-N分钟:])
    │
    ▼
计算波动率比率
vol_ratio_5_30 = vol_5m / vol_30m
vol_ratio_30_120 = vol_30m / vol_120m
```

```python
from tqdm import tqdm

def parallel_process(symbols: list, dates: list, max_workers: int = 4):
    """多进程并行处理"""
    tasks = [(s, d) for s in symbols for d in dates]

    with ProcessPoolExecutor(max_workers=max_workers) as executor:
        futures = {
            executor.submit(process_one, symbol, date): (symbol, date)
            for symbol, date in tasks

                if not ok:
```

## 附录A：数据采样时间规则详解

### A.1 为什么需要严格小于？

```
时刻T的真正含义是："在T时刻，我能看到什么数据"

交易所时间 timestamp=T 表示："这笔数据在T时刻产生"
local_timestamp=T 表示："这笔数据在T时刻到达我的机器"

如果我现在是时刻T，我能看到的最新数据应该是：
- timestamp < T （因为T时刻产生的数据在T时刻还不可见）
- local_timestamp < T （因为到达时间T的数据在T时刻还不可见）
```

### A.2 为什么同一timestamp要选最大local_timestamp？

```
情况：同一个timestamp有多个local_timestamp

timestamp=1000, local_timestamp=1001  ✓
timestamp=1000, local_timestamp=1002  ✓
timestamp=1000, local_timestamp=1003  ✓

这三个都是同一时刻产生的数据，但到达时间不同。
我们应该选择最新的那个（local_timestamp=1003），
因为它包含最完整的信息。
```
